# Helpdesk Escalation Matrix

| Issue Type | L1 (Helpdesk) | L2 (Senior IT) | L3 (Specialist) | Target Escalation Time |
|---|---|---|---|---|
| Password reset / account unlock | ✅ Resolve | — | — | N/A |
| Software installation | ✅ Resolve | If licence issue | — | N/A |
| Hardware fault | ✅ Diagnose | ✅ Arrange repair/replace | — | Same day |
| Network / VPN — single user | ✅ Resolve | If reinstall fails | Network Eng. | 2 hours |
| Network — multiple users | Triage only | Network Eng. | ISP/vendor | Immediate |
| Server / cloud outage | Triage only | — | Cloud/DevOps team | Immediate |
| Security incident | ✅ Contain device | ✅ Escalate | Security Operations | Immediate (P1) |
| New employee onboarding | ✅ Full ownership | — | — | N/A |

**Escalation principle:** Always tell the user when and why you are escalating. Never let a ticket sit escalated for >2 hours without an update to the user.
